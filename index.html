Hybrid Task Graph Scheduler API

The hybrid task graph scheduler (HTGS) API is designed to aid in creating task graphs for algoritms to obtain performance across CPUs and multiple-co-processors.

The source code for HTGS is found at HTGS GitHub Repository

Installation and Usage Instructions

Dependencies

  1. cmake version 2.7+ (optional: with cmake-gui or ccmake)
  2. g++/gcc version 4.8.4+ (must support C++11)
  3. pthreads
  4. (Optional) doxygen

Compile and Testing

  1. Clone the repository: 'git clone https://github.com/usnistgov/HTGS.git' or download from HTGS GitHub Repository
  2. Create a 'build' directory
  3. From the build directory configure HTGS build parameters: 'cmake-gui <HTGS_Directory>' (or ccmake)
    • CMAKE_INSTALL_PREFIX - Where to install HTGS and documentation
    • BUILD_DOXYGEN - Creates doxygen documentation, can be viewed online at: HTGS Documentation
    • RUN_GTEST - Compiles and runs google unit tests for HTGS (can be run using 'make run-test')
  4. Configure and Build
  5. Run 'make'
  6. Run 'make install'
  7. (Optional) To build documentation, run 'make doc'
  8. (Optional) To re-run GTEST, run 'make run-test'
